Build a complete lead-generation taxi website for:

Business Name: Lakeside & Purfleet Taxis Ltd
Location: Thurrock, Essex, UK
Website Type: Conversion-focused taxi enquiry website
Important: This is NOT a live booking/payment/dispatch website.

The website must have both:

1. Public front-end website
2. Admin back-end lead management system

The main goal is to capture taxi journey enquiries and allow the business owner to reply with pricing manually.

---

# Core Website Objective

Create a professional, non-generic, high-converting local taxi website that helps Lakeside & Purfleet Taxis generate more enquiries, phone calls, WhatsApp messages and local SEO traffic.

The website should not look like a typical AI-generated taxi website or SaaS landing page.

It should feel like a trusted local taxi company website with strong conversion strategy, local identity, premium layout, and clear customer journey.

Design direction:

* Premium taxi brand feel
* Black, yellow, white and subtle dark gradients
* Local Thurrock/Lakeside identity
* Strong hero sections
* Clear calls-to-action
* Mobile-first layout
* Trust-focused design
* Fast loading
* Professional but not corporate boring
* No generic stock-style layout
* No fake app booking interface
* No price calculator
* No live fare estimate

---

# Key Business Rule

The booking form must NOT:

* Show prices
* Calculate prices
* Take payment
* Confirm a live booking
* Connect to any dispatch system

The booking form must:

* Collect full journey details
* Show a final review/confirmation step
* Let the customer submit a booking/quote request
* Save the request as a lead in the admin panel
* Notify the business by email
* Allow the business owner/admin to reply manually with pricing
* Allow the admin to mark the lead as quoted, booked, completed, cancelled, or archived

Use language like:
"Request a Quote"
"Submit Booking Request"
"We'll confirm availability and price"
"Your booking is not confirmed until our team contacts you"

Do not use:
"Book Now" if it implies instant booking
"Pay Now"
"Fare Estimate"
"Instant Price"
"Confirmed Booking"

# Booking / Quote Request Form

Create a multi-step form.

## Step 1: Journey Type

Options:

* Local Taxi
* Airport Transfer
* School Run
* Corporate Travel
* Long Distance
* Other

## Step 2: Journey Details

Fields:

* Pickup location
* Destination
* Via stops / extra stops optional
* Journey date
* Journey time
* Return journey required? Yes/No
* If return yes: return date and return time

## Step 3: Passenger Details

Fields:

* Number of passengers
* Luggage details
* Child seats required? Yes/No
* Accessibility requirements optional
* Additional notes

## Step 4: Customer Details

Fields:

* Full name
* Mobile number
* Email address
* Preferred contact method:

  * Phone
  * WhatsApp
  * Email

## Step 5: Review & Submit

Display all submitted information clearly.

Include confirmation notice:
"This is a booking request, not a confirmed booking. Our team will review your journey details and contact you with price and availability."

Submit button:
"Submit Booking Request"

After submission:
Show thank-you page/message:
"Thank you. Your booking request has been sent to Lakeside & Purfleet Taxis. Our team will contact you shortly to confirm availability and pricing."

Admin can open each lead and:

* View all details
* Add internal notes

* Change lead status

* Mark as booked
* Add booking reference
* Archive lead

## Reply To Customer

Admin should be able to send an email reply from the lead detail page.

Email reply should include:

* Customer name
* Journey summary
* Quoted price entered by admin
* Message from admin
* Confirmation instructions
* Business contact details

Admin should have editable message box before sending.

Do not automatically confirm booking unless admin marks it as booked.

## Booking Management

When a lead is marked as booked, move/show it under Booked Jobs.

Booked job should include:

* Customer details
* Journey details
* Date/time
* Quoted price
* Booking status
* Admin notes
* Optional assigned driver
* Optional booking reference

This is only internal job management, not dispatch software.

Do not build live driver tracking.
Do not build fleet dispatch.
Do not build customer payment.
Do not build automatic price calculation.
